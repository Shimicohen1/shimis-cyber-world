Weak, never-expiring passwords are still the primary entry point for far too many Linux compromises. I've seen countless "sophisticated" attacks boil down to a simple brute-force against a user with "password123". It's embarrassing.

This isn't rocket science, folks. You need to enforce a minimum length and character complexity on all your Linux systems – Ubuntu, RHEL, doesn't matter. And for crying out loud, make them expire. Ninety days is a good baseline.

If you’re not doing this, you're practically holding the door open. Configure /etc/login.defs for PASS_MAX_DAYS 90 and pam_pwquality.so for minlen = 14 and minclass = 4. This is basic hygiene that still gets overlooked.

Stop leaving low-hanging fruit for attackers.

## The fix

```
# /etc/security/pwquality.conf

minlen = 14

minclass = 4

# /etc/login.defs

PASS_MAX_DAYS 90

PASS_MIN_DAYS 7

PASS_WARN_AGE 14
```

**Reference:** CIS Linux Benchmark 5.4.1
