You've got a Copilot Studio agent that can chat about company data. Great. But what happens when that agent can also make raw HTTP calls to *any* endpoint, internal or external? SSRF via your chatbot. I’ve seen it. It’s not pretty.

This isn't theoretical. Copilot Studio agents, if left unchecked, can be weaponized for Server-Side Request Forgery. They can probe internal networks, hit metadata endpoints, or exfiltrate data to attacker-controlled servers using the HTTP connector. This is a red alert if you ask me.

The fix is Power Platform DLP policies. Classify connectors into Business, Non-business, and Blocked data groups. Block sensitive connectors like HTTP requests – it's the most dangerous. Use endpoint filtering to allowlist specific SharePoint sites or APIs instead of blanket access.

Run Add-PowerAppsAccount, then Get-DlpPolicy | Select DisplayName, CreatedTime | Format-Table to see your current policies. If you don't have policies for Copilot Studio, you're leaving a massive door open.

## The fix

```
# List active DLP policies and their connector restrictions
Add-PowerAppsAccount
Get-DlpPolicy | Select DisplayName, CreatedTime | Format-Table
```
