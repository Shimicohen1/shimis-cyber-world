If an attacker gets a foothold on your Linux server and that server allows direct root SSH login, it's game over. Full system control from a single compromised password or key. This isn't theoretical; it's how major breaches pivot.

The fix is ridiculously simple, yet I still see far too many organizations leaving this door wide open. Don't be one of them. Prevent direct SSH login as root. Force attackers to work harder for privilege escalation.

Edit your sshd_config and set: PermitRootLogin no. Then, a quick restart: sudo systemctl restart sshd. This forces users to log in with a standard account and then use sudo for privileged actions. Basic hygiene, massive impact.

Go check your Linux servers now. This is non-negotiable.

## The fix

```
# /etc/ssh/sshd_config

PermitRootLogin no

# Then restart:

sudo systemctl restart sshd
```
