If an attacker gets root on your system, it’s game over. Full stop. The number of times I’ve seen this exact scenario play out, where a single compromised password hands over the keys to the kingdom, is frankly alarming.

Direct root SSH login is a security relic. It's an open invitation for brute-force attacks and a massive privilege escalation shortcut for anyone who snags a root credential, however they got it. Remember Codecov? SSH keys were the pivot.

The fix is simple, non-disruptive, and mandatory: disable direct root SSH access. Use sudo for privilege escalation after logging in with a regular user account.

Edit /etc/ssh/sshd_config and set:
PermitRootLogin no

Then restart the SSH service:
sudo systemctl restart sshd

This basic hardening measure significantly raises the bar for attackers. If you're not doing this, you're leaving a critical door wide open.

Take five minutes and implement this now.

## The fix

```
# /etc/ssh/sshd_config

PermitRootLogin no

# Then restart:

sudo systemctl restart sshd
```
