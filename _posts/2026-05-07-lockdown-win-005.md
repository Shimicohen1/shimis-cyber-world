WannaCry, NotPetya, EternalBlue. These aren't ancient history; they're a stark reminder of what happens when you leave an ancient, vulnerable protocol like SMBv1 enabled. It's a gaping hole.

SMBv1 has no place in a modern network. It's obsolete, insecure, and a favourite target for attackers. Even patched systems are at risk from new zero-days because the underlying protocol is fundamentally flawed. If you're running Windows Server, this is a non-negotiable step.

The fix is simple and immediate. Get rid of it.

Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
Disable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol -NoRestart

Don't wait for the next ransomware wave to force your hand. This is basic hygiene that far too many organizations still overlook.

## The fix

```
# PowerShell

Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force

# Disable SMBv1 client

Disable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol -NoRestart
```
